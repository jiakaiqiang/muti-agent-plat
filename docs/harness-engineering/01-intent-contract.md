# 01 Intent Contract 需求契约

## 目的

把用户自然语言需求转成可交接、可评审、可验收的需求契约。

## 参与角色

- 用户
- Requirement Agent
- Review Agent

## 准入条件

- 已获得用户原始需求。
- 已知道本次需求所属的大致项目或能力域。

## 输入

- 用户原始需求
- 已知背景
- 历史偏好或约束

## 输出

- 目标
- 背景
- 非目标
- 约束
- 验收标准
- 风险
- 需要用户确认的问题

## 退出条件

- 目标明确。
- 非目标明确。
- 验收标准可以被判断。
- 风险已列出。
- 待确认问题已解决，或已进入人工确认。

## 不允许

- 不允许直接进入实现。
- 不允许把技术方案写成需求。
- 不允许使用“体验更好”“尽量稳定”这类不可判断的验收标准。

## 返工条件

- 设计阶段发现目标不清。
- 评审阶段发现验收标准不可判断。
- 用户改变范围。
- 实现阶段发现需求与实际约束冲突。
