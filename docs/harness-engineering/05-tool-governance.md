# 05 Tool Governance 工具治理

## 目的

规定 Agent 使用工具、修改文件、执行命令、访问外部资源时的工程边界。

## 风险分级

### 低风险

默认允许。

- 读取文件
- 搜索代码
- 查看文档
- 总结已有内容

### 中风险

允许，但必须留痕。

- 在允许范围内修改代码
- 运行本地构建
- 运行本地验证命令
- 写入本地交付产物

### 高风险

必须人工确认。

- 删除文件
- 越界修改文件
- 调用真实外部 API
- 写生产数据
- 发送外部通知
- 使用真实密钥或真实模型产生费用

## 阶段规则

Requirement / Design：

- 以读取和分析为主。
- 不应写代码。

Planning：

- 必须声明后续任务可能需要的工具风险。

Implementation：

- 只能在任务允许范围内修改。
- 范围内写入属于中风险，必须留痕。
- 越界写入属于高风险，必须确认。

Verification：

- 可以运行必要验证命令。
- 破坏性命令或外部调用必须确认。

Delivery：

- 可以生成草稿。
- 外发、写外部系统必须确认。

## 硬规则

- 高风险动作不能由 Agent 自行决定。
- 权限不足时不能绕过，只能进入 Human Intervention。
- 工具使用必须能在最终交付中追溯。
