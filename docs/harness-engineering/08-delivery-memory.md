# 08 Delivery Memory 交付记忆

## 目的

规定一次交付结束后，哪些经验需要沉淀为项目知识。

## 需要沉淀的内容

### 项目知识

- 本次需求属于哪个能力域。
- 涉及哪些代码或文档范围。
- 哪些模块边界被确认。

### 设计决策

- 采用了什么方案。
- 放弃了什么方案。
- 为什么这样选择。

### 失败模式

- 哪些问题导致返工。
- 哪些上下文不足。
- 哪些权限或工具边界需要注意。

### 验收经验

- 哪些证据能有效证明交付满足需求。
- 哪些验证方式不可靠。

### 用户偏好

- 用户明确表达的偏好。
- 用户确认过的约束。

## 写入规则

- 只有对后续任务有复用价值的内容才沉淀。
- 不把一次性细节写成长期知识。
- 不重复写入已有知识。
- 不确定是否长期有效时，只留在本次交付记录中。

## 退出条件

- 最终交付说明了哪些内容被沉淀。
- 沉淀内容能帮助后续 Agent 更快理解项目。
- 没有把噪音、临时决策或过期信息写成长期记忆。
