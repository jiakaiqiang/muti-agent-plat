# 06 Human Intervention 人工干预

> 最后修改时间：2026-06-11 14:47:47 +08:00
> 修改人：Codex
> 修改的 Agent：Codex

## 目的

规定什么时候必须让用户或人工决策介入。

## 触发条件

必须人工干预的情况：

- 需求目标不清楚。
- 验收标准无法判断。
- 设计存在多个高影响方案。
- 需要真实密钥、真实模型或真实外部 API。
- 需要删除、覆盖或越界修改重要文件。
- 需要外发通知或写外部系统。
- 评审发现范围发生变化。
- 同一问题多次返工仍不收敛。

## 人工确认记录

每次人工干预必须记录：

- 干预原因
- 需要确认的问题
- 可选方案
- 用户或人工选择
- 选择理由
- 确认后回到哪个阶段

## 不允许

- 不允许把人工确认只留在聊天里。
- 不允许 Agent 猜测用户意图后继续推进。
- 不允许用默认选项绕过高风险确认。

## 退出条件

- 需要确认的问题已获得明确选择。
- 后续阶段知道应该如何继续。
- 如果无法确认，交付应暂停或失败，而不是继续冒进。
